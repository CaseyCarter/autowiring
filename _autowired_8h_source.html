217  // !!!!! READ THIS IF YOU ARE GETTING A COMPILER ERROR HERE !!!!!
218  // If you are getting an error tracked to this line, ensure that class T is totally
219  // defined at the point where the Autowired instance is constructed. Generally,
220  // such errors are tracked to missing header files. A common mistake, for instance,
221  // is to do something like this:
222  //
223  // class MyClass;
224  //
225  // struct MyStructure {
226  // Autowired<MyClass> m_member;
227  // };
228  //
229  // At the time m_member is instantiated, MyClass is an incomplete type. So, when the
230  // compiler tries to instantiate AutowiredCreator::Create (the function you're in right
231  // now!) it finds that it can't create a new instance of type MyClass because it has
232  // no idea how to construct it!
233  //
234  // This problem can be fixed two ways: You can include the definition of MyClass before
235  // MyStructure is defined, OR, you can give MyStructure a nontrivial constructor, and
236  // then ensure that the definition of MyClass is available before the nontrivial
237  // constructor is defined.
238  //
239  // !!!!! READ THIS IF YOU ARE GETTING A COMPILER ERROR HERE !!!!!
